centos编译mldonkey解决中文乱码

mldonkey可能是最好最稳定的p2p下载软件了，除了支持eD2k和Overnet两种网络外还可以支持BT,DC,Gnutella,Fasttrack,Kad等多种流行的P2P应用网络。编译好的mldonkey-3.1.0.static.i386-Linux_glibc-2.7.tar.bz2汉字文件名乱码。下载mldonkey-3.1.2.tar.bz2源代码编译解决：

tar xfj mldonkey-3.1.2.tar.bz2 -C /usr/local/src/

cd /usr/local/src/mldonkey-3.1.2/

export LANG=zh_CN.UTF-8

export LC_ALL=zh_CN.UTF-8

./configure

gmake &amp;&amp; make install

启动脚本

cat mldonkey.sh 

#!/bin/sh

#

PATH=/bin:/sbin:/usr/bin:/usr/sbin:/usr/local/mysql/bin

export LANG=zh_CN.UTF-8

export LC_ALL=zh_CN.UTF-8

KILL=/bin/kill

KILL_OPT=-9

command=&quot;/usr/local/bin/mlnet&quot;

stop_service()

{

&nbsp;&nbsp;&nbsp; pid_of_mlnet=`pidof mlnet`

&nbsp;&nbsp;&nbsp; $KILL $KILL_OPT&nbsp; $pid_of_mlnet

&nbsp;&nbsp;&nbsp; pid_of_mlnet=`pidof mlnet`

&nbsp;&nbsp;&nbsp; if [ $pid_of_mlnet ]; then

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; echo &quot;unable to stop mlnet.try do it mannually.&quot;

&nbsp;&nbsp;&nbsp; else

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; echo DONE

&nbsp;&nbsp;&nbsp; fi

}

start_service()

{

&nbsp; ${command} &amp;

}

case $1 in

&#39;start&#39;)

&nbsp;&nbsp;&nbsp; start_service

&nbsp;&nbsp;&nbsp; ;;

&#39;stop&#39;)

&nbsp;&nbsp;&nbsp; stop_service

&nbsp;&nbsp;&nbsp; ;;

&#39;restart&#39;)

&nbsp;&nbsp;&nbsp; stop_service

&nbsp;&nbsp;&nbsp; start_service

&nbsp;&nbsp;&nbsp; ;;

*)

&nbsp;&nbsp;&nbsp; echo &quot;$0:&nbsp; usage:&nbsp; $0 {start|stop|restart}&quot;

&nbsp;&nbsp;&nbsp; ;;

esac

&nbsp;

以上在centos linux中测试通过。

gpt分区不能用物理硬盘创建pv

LUN只是一个逻辑的东西。在存储设备内部，LUN与物理硬盘的关系，如同主机中逻辑卷与物理硬盘的关系，也就是好像LV和PV的关系，在不同范畴。可以理解为LUN就是在做完RAID之后的逻辑盘上分出来的logic driver。但不同的是，LUN在主机上看来是一个物理硬盘。

一个硬盘在能够被LVM使用之前一定要初始化，可以使用pvcreate命令将PVRA的信息写入到硬盘当中，而这样被写入了PVRA信息的硬盘，就叫做PV。

&nbsp;&nbsp;&nbsp;&nbsp; 注意：如果是启动盘，还应该加上-B的选项。这样做将会在硬盘头保留2912KB的空间给LVM表头（LVM header），相关的信息，可以参考LVM的结构信息。如果你想要做启动盘的镜像的话，可以参考根盘镜像。

&nbsp;&nbsp;&nbsp;&nbsp; 将硬盘初始化以后，就可以将该PV添加到已经存在的VG当中去了。

&nbsp;&nbsp;&nbsp;&nbsp; 对于磁盘阵列，在硬盘初始化的时候，使用的设备是&ldquo;rdsk&rdquo;，而在将PV添加到已经存在的VG的过程中，使用的设备是&ldquo;dsk&rdquo;。对于&ldquo;rdsk&rdquo;和&ldquo;dsk&rdquo;的区别，应该有非常清晰的认识，如果觉得不清楚，可以查看相关资料。以下操作在centos 6测试通过。

nice调整SSHD的启动优先级别

centos linux服务器远程连接无响应，有可能是由于apache或者是mysql等服务过载，或者受到dos等攻击造成服务器无响应。

为避免强制重启，可以调高sshd优先级。使用-2优先级是希望sshd服务的优先级高于apache等的0，但低于log等的-5。

要运行的命令作为 nice 的一个参数。而其另一个参数 -n 可用来设置优先级别。默认情况下 nice 将设置优先级为 10。

查看启动脚本

vim /etc/init.d/sshd 

修改里面的

$SSHD $OPTIONS &amp;&amp; success || failure

为

nice -n -2 $SSHD $OPTIONS &amp;&amp; success || failure

或者

sed -i &#39;s/$SSHD $OPTIONS/nice -n -2 $SSHD $OPTIONS/&#39; /etc/init.d/sshd

sudo /etc/init.d/sshd restart

查看优先级

ps -aeo cmd,pid,nice |grep sshd

系统中运行的每个进程都有一个优先级(亦称“nice 值”)，其范围从 -20 (最高优先级)到 19 (最低优先级)。默认情况下，进程的优先级是 0 (“基本”调度优先级)。优先级比较大的进程(nice 值比较小，最低到 -20)相对优先级比较小的进程(直到 19)将比较频繁地被调度运行，因此就拥有更多的进程周期。一般用户只能降低它们自己进程的优先级别，并限于 0 到 19 之间。超级用户(root)可以将任何进程的优先级设定为任何值  

如果某个或某些进程消耗了过多的系统资源，除了终止它们，您还可以更改它们的优先级。为此，请使用 renice 命令。其语法为：

renice 优先级 [[-p] pid ...] [[-g] pgrp ...] [[-u] 用户名 ...]

DNS劫持与RBL使用的解决办法

  使用opendns nameserver 208.67.220.220 208.67.222.222可以解决dns hijacking，缺点是web浏览时可能不会解析到最快的ip地址。电信、联通nameserver 如202.106.196.115 202.106.0.20 202.106.46.151多被劫持。导致邮件服务器使用rbl服务阻挡全部邮件。解决办法之一是用返回码验证。postfix修改main.cf：

  reject_rbl_client cblless.anti-spam.org.cn=127.0.8.5,

&nbsp; reject_rbl_client zen.spamhaus.org=127.0.0.2,</span><span class="body"> </span>

…

关于RBL的原理。目前用于垃圾邮件过滤的RBL服务，应该称之为基于DNS的实时黑名单查询，也就是说，这个服务是通过DNS协议来完成的。

　　具体而言，当一个客户端希望查询某个IP地址（如1.2.3.4）是否在某个RBL（如cbl.anti-spam.org.cn）中 是，其实际上是查询如下地址是否存在解析： 4.3.2.1.cbl.anti-spam.org.cn. （IP地址逆转附加在RBL地址后）。DNS的解析分为几种类型，对于RBL查询，通常是查询这个地址是否存在A记录、TXT记录或者任意（ANY）记 录。

　　如果该地址被列入了这个RBL，那么查询会返回一个具体的解析结果，根据RBL和查询的不同，可以返回一段文本，也可以返回一个或几个IP地 址，也可以同时返回文本和IP。返回的文本通常是一个说明，用来说明这个IP地址被列入了哪个RBL，具体信息去哪里查询等。返回的IP地址并不具有实际 意义，只是标识该查询有结果，通常这个IP地址是一个保留IP段的地址，如127.0.0.1、127.0.0.2等。

　　如果该地址没有被列入这个RBL，那么该查询会返回一个查询错误（NXDOMAIN），表示该地址未列入。
