用u盘/移动硬盘升级pc服务器bios

pc服务器，又称工业标准服务器。升级bios微码一般需要软盘或光盘。软盘难以找到，刻盘也麻烦。就考虑用u盘/移动硬盘升级bios，以下在ibm x-366服务器（m/t m8836-isx）测试通过。

准备工作：

制作可引导移动硬盘，参考http://veryi.com/w/257.html；

下载bios镜像，复制到移动硬盘相应目录。

编辑menu.lst，添加：

title bios CD ISO

&nbsp; fallback 8

&nbsp; find --set-root /bios/ibm_fw_bios_zue166a_anyos_i386.iso

&nbsp; map /bios/ibm_fw_bios_zue166a_anyos_i386.iso (0xff) || map --mem /bios/ibm_fw_bios_zue166a_anyos_i386.iso (0xff)

&nbsp; map --hook

&nbsp; chainloader (0xff)

&nbsp; savedefault --wait=2

title bios DOS IMG

&nbsp; find --set-root --ignore-floppies /bios/ibm_fw_bios_zue166a_anyos_i386.img

&nbsp; map --mem /bios/ibm_fw_bios_zue166a_anyos_i386.img (fd0)

&nbsp; map --hook

&nbsp; chainloader (fd0)+1

&nbsp; rootnoverify (fd0)

文件名和目录要对应。

升级过程：

要保证供电稳定，启动服务器，按F12，选usb启动。然后选择cd iso或dos img加载。后面步骤参考

http://www-900.ibm.com/cn/support/viewdoc/detail?DocId=QFAN-7UK6DK

AMD X4 631 必须使用独立显卡

    一台新组装的pc机开机黑屏不显示，不报警，也没有滴的一声。配置为 AMD Athlon(tm) II X4 631 Quad-Core Processor，asus F1A55-M LX主板、A55 FCH（Hudson D2）芯片组。想当然认为是主板或cpu有问题。看到cpu盒子上写着“需要独立显卡-Requires Discrete GPU”，最后终于明白，这种主板上只是按照了vga的接口，而没有集成显示控制芯片，631又屏蔽了内部集成的显卡，不安装独立显卡，系统当然不会显示。

windows 7设置静态ip地址不生效

  windows 7设置静态ip地址不生效，ipconfig显示为一个以169开头的ip，开始以为是设置方法不对。最后找到原因：ip地址冲突，手工设定的ip地址不生效。系统不提示ip地址冲突，机子就自己给自己设置一个169.254..的ip。所以用了不少时间查找原因。

chkconfig --list dhcpd

centos 6 启动eth0 网卡时报错：

Error: Connection activation failed: Device not managed by NetworkManager

1\. Remove Network Manager from startup Services.

chkconfig NetworkManager off

chkconfig --list NetworkManager

2\. Add Default Net Manager

chkconfig network on

chkconfig --list network

Stop NetworkManager first

service NetworkManager stop

and then start Default Manager

service network start

or you can change it easy:

system-config-network 

uncheck the box that sets the eth0 as managed by NetworkManager and set IP info from there.

system-config-services 

stop and disable NetworkManager , start and enable network

you have problem because both of NetworkManager and Network using same Network driver and default network blocking access for NetworkManager. if you disable NetworkManager your problem will disappeared after restart.

squid Version 3.2.0.13: ERROR: No

forward-proxy ports configured.

&gt; #http/https ports

&gt; http_port 33070 transparent

&gt;

So, only one port, and its a NAT traffic receiving port. Looks like 

Squid was right, &quot;no forward-proxy ports configured&quot;.

Should not be on every request, that is a sign of some other problem 

happening. Just on the ones which need to produce internal URLs (ie 

error page icons, FTP directory icons, cache manager requests, etc). 

Squid requires a forward-proxy port for the URL it tells the browser to 

fetch those icons from. A NAT or TROXY receiving port is somewhat 

vulnerable to forgery attacks and Squid now avoids publishing its 

details. Accel ports have some problematic cases in the parser and until 

those are figured out properly they are avoided as well. Which just 

leaves the regular old forward-proxy port for now.

Adding &quot;http_port 3128&quot; or somesuch is enough to avoid this message

板载raid1安装centos 6 x86_64 linux系统

由于centos 6标准内核不包含raid驱动，需要单独安装或者在安装是加载，服务器是惠普ProLiant DL120 G7，阵列是HP Smart Array B110i SATA-3G RAID控制器（RAID 0/1/1+0）,。

进到BIOS里配置好raid。

在hp网站下载

Driver Diskette for HP Smart Array B110i SATA RAID Controller for Red Hat Enterprise Linux 6 (AMD64/EM64T) 。

输出到u盘上

For a USB key for Red Hat and SuSE

&nbsp;&nbsp;&nbsp; dd if=hpahcisr-1.2.6-7.rhel6.x86_64.dd&nbsp; of=/dev/sd?

解压出了一个dd.img文件

At boot prompt type &ldquo;linux nostorage&rdquo;

光驱放入安装光盘，在安装界面的第一个画面输入 ：linux nostorage&nbsp; （用linux text dd 不成功）

安装程序启动，会提示是否需要加载，按Yes，在弹出的对话框里找到含有驱动的USB设备，选中拷贝好的.img文件，确定，驱动加载完毕，安装程序提示是否需要继续加载其他驱动，如无其他设备，选择No，返回安装界面，按正常步骤操作即可，安装过程中会发现安装程序能正常识别到raid驱动设备了；

接下来，将U盘拔出，因为U盘符占用/dev/sda,不拔出linux会把U盘当硬盘的，这样系统直至安装结束。

yum -y update 升级，如果升级了内核，新的内核不会自动添加raid驱动的，重启后就进不了系统了，所以升级完毕切记不要忙着重启 , 以CentOS升级到6.0举例：

b110i.ko 是驱动模块，如果更新前的内核里不包含这个文件可以在驱动的.img文件里找到：

&nbsp;

mkdir ~/b110i&nbsp; 

mkdir /mnt/tmp&nbsp; 

mount -o loop dd.img /mnt/tmp&nbsp; 

cd /mnt/tmp&nbsp; 

cp modules.cgz ~/b110i&nbsp; 

cd ~/b110i&nbsp; 

gunzip -S .cgz modules.cgz&nbsp; 

cpio -ivd &lt; modules&nbsp; 

cp 2.6.18-194.el5/x86_64/b110i.ko /lib/modules/2.6.18-238.12.1.el5/updates/b110i.ko #注意根据实际版本情况适当调整&nbsp; 

depmod -v 2.6.18-238.12.1.el5&nbsp; 

mv /boot/initrd-2.6.18-238.12.1.el5.img /boot/initrd-2.6.18-238.12.1.el5.img.backup&nbsp; 

mkinitrd /boot/initrd-2.6.18-238.12.1.el5.img 2.6.18-238.12.1.el5 

参考http://coralzd.blog.51cto.com/90341/580046

Linux下源代码编译安装MySQL 5.5.16完整安装教程

以下安装在centos5.6、Yellow Dog Linux release 6.2 (Pyxis)通过测试。

MySQL 最新的版本5.5.16需要cmake编译安装

1&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; mysql 5.5.16编译安装

安装所需要系统库相关库文件

yum install -y gcc gcc-c++ gcc-g77 autoconf automake zlib* fiex* libxml* ncurses-devel libmcrypt* libtool-ltdl-devel*

#yum install -y cmake

创建mysql安装目录和数据存放目录　

mkdir -p /usr/local/mysql/

mkdir -p /opt/data/mysql/

创建用户和用户组与赋予数据存放目录权限

groupadd -g 502 mysql

useradd -u 502 -g mysql mysql

chown mysql.mysql -R /opt/data/mysql/

安装cmake

（mysql5.5以后是通过cmake来编译的）

下载解压cmake-2.8.5.tar.gz

wget http://www.cmake.org/files/v2.8/cmake-2.8.5.tar.gz

tar zxf cmake-2.8.5.tar.gz -C /usr/local/src

cd /usr/local/src/cmake-2.8.5

./configure

make -j6 &amp;&amp; make install

安装mysql 5.5.16

wget http://mirrors.sohu.com/mysql/MySQL-5.5/mysql-5.5.16.tar.gz

tar zxf mysql-5.5.16.tar.gz -C /usr/local/src

cd /usr/local/src/mysql-5.5.16

编译mysql-5.5.16

CFLAGS=&quot;-O3&quot; CXX=gcc

CXXFLAGS=&quot;-O3 -felide-constructors -fno-exceptions -fno-rtti&quot;

/usr/local/bin/cmake -DCMAKE_INSTALL_PREFIX=/usr/local/mysql \

-DMYSQL_UNIX_ADDR=/tmp/mysql.sock \

-DDEFAULT_CHARSET=utf8 \

-DDEFAULT_COLLATION=utf8_general_ci \

-DWITH_EXTRA_CHARSETS:STRING=all \

-DWITH_MYISAM_STORAGE_ENGINE=1 \

-DWITH_INNOBASE_STORAGE_ENGINE=1 \

-DWITH_READLINE=1 \

-DENABLED_LOCAL_INFILE=1 \

-DMYSQL_DATADIR=/opt/data/mysql \

-DMYSQL_TCP_PORT=3306

#-DWITH_MEMORY_STORAGE_ENGINE=1 \

#-DMYSQL_USER=mysql \

-DCMAKE_INSTALL_PREFIX=/home/mysql/mysql -DEXTRA_CHARSETS=all

CMake Warning:

&nbsp; Manually-specified variables were not used by the project:

&nbsp;&nbsp;&nbsp; MYSQL_USER

&nbsp;&nbsp;&nbsp; WITH_MEMORY_STORAGE_ENGINE

nohup make -j6 &amp;&amp; make install

复制配置文件

mv /etc/my.cnf /etc/my.cnf-`date +%F`

mkdir -p /usr/local/mysql/etc

cp support-files/my-medium.cnf /usr/local/mysql/etc/my.cnf

cd /usr/local/mysql/

mkdir log run tmp data

chgrp -R mysql.mysql *

cd /opt/data/mysql/

chown mysql.mysql /opt/data/mysql/

vim etc/my.cnf

innodb_log_files_in_group=2

innodb_data_home_dir = /opt/data/mysql/

innodb_data_file_path = ibdata1:2000M;ibdata2:2000M;ibdata3:20M:autoextend

innodb_log_group_home_dir = /opt/data/mysql/

# 4G RAM

innodb_buffer_pool_size = 1G

innodb_log_file_size = 256M

innodb_log_buffer_size = 8M

innodb_flush_log_at_trx_commit=0

innodb_thread_concurrency=8

innodb_flush_method=O_DIRECT

# perform

tmp_table_size = 512M

max_heap_table_size=128M

初始化数据库

(执行前 需要chmod 755 scripts/mysql_install_db 赋给文件执行权限)

cd /usr/local/src/mysql-5.5.16

sh ./scripts/mysql_install_db&nbsp; --user=mysql&nbsp; --basedir=/usr/local/mysql --datadir=/opt/data/mysql/

注：#basedir mysql安装路径 datadir 数据库文件储存路径

设置mysqld的开机启动:

cp support-files/mysql.server /etc/init.d/mysqld

chmod 755 /etc/init.d/mysqld

chkconfig mysqld on

chkconfig --list mysqld 

配置环境

#加入mysql到路径

echo pathmunge /usr/local/mysql/bin after &gt; /etc/profile.d/mysql.sh

#执行一下，保证mysql在路径环境变量中

. /etc/profile

或者

vim /etc/profile

PATH=$PATH:/usr/local/mysql/bin

PATH=/usr/local/mysql/bin:$PATH

启动mysql服务

/etc/init.d/mysqld start&nbsp; //服务器端开关

启动完成之后用ps -ef |grep mysql 命令查看是否启动
