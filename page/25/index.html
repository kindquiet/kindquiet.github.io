u盘/移动硬盘（usb hd）安装多种linux live CD

如果U盘够大，可以在一个U盘安装slax,pclinuxos,Linux Mint,puppy Linux等多种live linux。移动硬盘原理相同，不做区分。

因为U盘常用来拷贝文件，fat格式在多种操作系统上通用，所以用grub4dos管理U盘引导。

把U盘格成HDD格式：

用DiskGenius 3.2把U盘格式化为USB-HDD格式，推荐使用FAT32分区；或者用U盘格式化工具（HPUSBFW.EXE ）格式化为 HDD 格式。还有其它 工具如FlashBoot、 WIN PM 等。

将引导代码写入U盘mbr：

Win32下，运行grubinst_gui.exe ，在&ldquo;磁盘&rdquo;处选中U盘，在&ldquo;选项&rdquo;区选中&ldquo;不保存原来的mbr&rdquo;、&ldquo;不引导原来的mbr&rdquo;、&ldquo;启动时不搜索软盘&rdquo;、&ldquo;输出详细信息&rdquo;，点击&ldquo;安装&rdquo;即可。

拷贝grub4dos的grub.exe、grldr、menu.lst到u盘根目录。

现在，U盘已经可以用grub引导，把电脑设置为U盘优先引导，测试一下grub的引导。

安装linux live cd的过程分为两步，一是把live cd的iso镜像文件中的相关目录和文件复制到U盘，可以用Virtual CloneDrive加载iso镜像。二是编辑grub的menu.lst，各个版本的linux差别较大。

安装Linux slax

提取slax-6.1.2.iso中的

boot

slax

目录（包含文件，其实boot目录只需vmlinuz 、initrd.gz即可）到U盘根目录即可，这个是最容易的安装的linux live之一，而且slax启动速度快，对声卡等硬件支持更好，至少在我测试过的live cd linux中如此。

grub菜单：

title Slax

&nbsp;&nbsp;&nbsp; root (hd0,0)

&nbsp;&nbsp;&nbsp; kernel /boot/vmlinuz root=/dev/ram0 rw max_loop=255 init=linuxrc load_ramdisk=1 ramdisk_size=6666 slax from=/slax/ vga=normal autoexec=startx

&nbsp;&nbsp;&nbsp; initrd /boot/initrd.gz

&nbsp;&nbsp;&nbsp; boot

&nbsp;&nbsp;&nbsp; initrd /isolinux/initrd.gz

安装archinux（未成功）

grub菜单：

ERROR:boot device didn't show up for 30 seconds

在 ramfs$ 提示符下依次输入:

ln -s /dev/sdb1 /dev/archiso

exit

假定 sdb1 是U盘的挂载位置。

这个问题仅在U盘启动的时候产生，更准确的说，是仅在没有使用 dd 命令制作的U盘启动盘上发生。

为什么呢？因为这种情况下，U盘的标签（Label）与U盘上 boot/grub/menu.lst 中指定的不相符了。

因此，一个更好的解决办法是，修改U盘上 boot/grub/menu.lst :

kernel /boot/vmlinuz26 lang=en locale=en_US.UTF-8 archisolabel=ARCHISO_OINGAE2I ramdisk_size=75%

将 &ldquo;ARCHISO_OINGAE2I&rdquo; 改为你U盘的标签（Label）。是什么就改成什么，注意，要区分大小写。存盘即可。

百忍成金

比如听到有人骂我，高高兴兴听着，不生气，让他骂一百次。

忍无可忍

  东张公五世同居，百忍家道兴。据说他发愿，在他的一生中，要行一百件大忍辱的事，忍过了九十九次之后，第一百次时是他的孙子娶妻那一天，突然来了个道人， 要试验他是否真有忍辱功夫，便向他要这个新娘子，先于自己做一夜夫妻，这一件事使张公很感为难，但他宽大一想，我什么悔辱事都忍受过了，这最后一次忍辱有 什么不能呢？

linux系统apache tomcat虚拟主机安装配置

  多年不用tomcat，应一朋友要求，搭建一个java环境的虚拟主机，经过多次测试，现在把可以正常运行的配置记录下来。

安装jdk

cd /usr/local

sh /mnt/mysql/jdk-6u14-linux-i586.bin 

ln -s jdk1.6.0_14/ jdk

echo &quot;export JAVA_HOME=/usr/local/jdk

export PATH=$JAVA_HOME/bin:$PATH

export CLASSPATH=$CLASSPATH:$JAVA_HOME/lib&quot; &gt;&gt; /etc/profile.d/java.sh

chmod +x /etc/profile.d/java.sh

安装tomcat

wget http://apache.freelamp.com/tomcat/tomcat-6/v6.0.20/bin/apache-tomcat-6.0.20.tar.gz

tar xzf apache-tomcat-6.0.20.tar.gz -C /usr/local/

cd /usr/local/

ln -s apache-tomcat-6.0.20/ tomcat

vi ~/.bash_profile

TOMCAT_HOME=/usr/local/tomcat

export PATH TOMCAT_HOME

source ~/.bash_profile

安装jk connector

wget http://labs.xiaonei.com/apache-mirror/tomcat/tomcat-connectors/jk/source/jk-1.2.28/tomcat-connectors-1.2.28-src.tar.gz

tar xfz tomcat-connectors-1.2.28-src.tar.gz -C /usr/local/src/

cd /usr/local/src/tomcat-connectors-1.2.28-src/

cd native/

./configure --with-apxs=/usr/local/httpd/bin/apxs

make

cp apache-2.0/mod_jk.so /usr/local/httpd/modules/

为喜欢旅行和骑车上班的朋友推荐一款音箱-轻骑兵U2使用感受

耳机时间长了会很不舒服，也会损伤听力。所以考虑买一个便携音箱。

轻骑兵U2迷你音箱的特点:

1、续航时间长，我每天上下班骑车2个小时，4节5#2000mah充电电池，已经听了一个月了。

2、声音够大，可以盖过路边汽车的噪音，额定功率2W比某款号称额定功率2.5W&times;2的声音大多了。

3、体积小，也就一个半拳头大，携带方便，适合骑车旅行、步行旅游者。

轻骑兵U2迷你音箱是轻骑兵公司专为户外运动爱好者量身定做一款产品。另外，轻骑兵U2还采用了主副箱设计，音乐回放时它的立体声输出分离度较高，整体性能在同级音箱中非常突出。同时，轻骑兵U2迷你音箱还可以作为手电筒照明使用，对于户外爱好者来说使用起来非常方便。

对于&ldquo;驴友&rdquo;来讲，随身装备必须做到精而又精，尤其音响设备要做到全天候，太重的音箱不会考虑，太娇贵的器材同样必须放弃。轻骑兵U2由铝合金及ABS工程塑料制成，强度高、重量轻，适合各种严酷的户外环境使用。

&nbsp;&nbsp;&nbsp; U2的造型比较独特，酷似一片树叶，又有点像水滴，充满了动感，切合它户外音箱的定位。

&nbsp;&nbsp;&nbsp; 供电方面轻骑兵U2使用4节5号电池，连续播放时间长达50小时左右。

&nbsp;&nbsp;&nbsp; 随机附带一根音箱连接线，3.5mm插头的音频输入线长短各一根，可方便接驳各种随身听及笔记本。

&nbsp;&nbsp;&nbsp; U2的输出功率为2W，实际听感音色嘹亮激越，金属质感极强，对于户外用户已经足够。

&nbsp;&nbsp;&nbsp; U2由于体积小巧，还可以当笔记本专用音箱，随机附送电源适配器，用户可选配USB供电线。

　　值得一提的是U2还内置一颗高亮度的白色LED，在暗夜还能照明用。附送红色防震袋，保证了音箱不被摔坏。

mysql 5.1.39，建立一个存储过程

BEGIN

    IF ISNULL(s) THEN

        RETURN ‘’;

    ELSEIF n<15 THEN

        RETURN LEFT(s, n);

    ELSE

        IF CHAR_LENGTH(s) <= n THEN

            RETURN s;

        ELSE

            RETURN CONCAT(LEFT(s, n-10), ‘ … ‘, RIGHT(s, 5));

        END IF;

    END IF;

END

ERROR 1418 (HY000): This function has none of DETERMINISTIC, NO SQL, or READS SQL DATA in its declaration and binary logging is enabled (you might want to use the less safe log_bin_trust_function_creators variable)

 

解决:

mysql> set global log_bin_trust_routine_creators=1;
